\section{Motivation}
\begin{itemize}
  \item \ac{PLM}-based services (ESM2, ProtT5) are increasingly adopted in computational biology research
  \item Biocentral is a published, production-grade platform for embedding-driven protein analysis (Franz et al., JMB 2026)
  \item Traditional software testing assumes deterministic outputs and known expected values --- neither holds for ML embedding systems
  \item Key challenges: no ground-truth oracle for embeddings, GPU non-determinism (floating-point), expensive compute for CI, model opacity
  \item Testing these systems is essential but under-explored compared to model development
\end{itemize}

\section{Problem Statement}
\begin{itemize}
  \item Biocentral server has 8 API modules, GPU-accelerated inference (Triton), async task pipelines, and database-backed caching
  \item Ensuring correctness across this stack requires a multi-layer test strategy
  \item Property-based and metamorphic tests can validate ML behavior without ground truth, but are computationally expensive
  \item Core tension: thorough testing vs.\ fast CI feedback loops
\end{itemize}

\section{Research Question}
\begin{itemize}
  \item RQ: \emph{What is the trade-off between CI execution time and test coverage for property-based testing in embedding services?}
  \item Sub-questions: Which test layers provide the most fault-detection value per unit of CI time? Can GPU-free deterministic embedders replace real models in CI without sacrificing coverage?
\end{itemize}

\section{Contributions}
\begin{itemize}
  \item A multi-layer test pyramid for embedding-driven ML services: unit $\to$ integration $\to$ property/oracle $\to$ performance $\to$ metamorphic experiments
  \item \texttt{FixedEmbedder}: a SHA-256-based deterministic embedder enabling the entire CI pipeline to run without GPUs
  \item 8 metamorphic experiments validating PLM embedding behavior (X-masking, reversed sequences, AA mutation sensitivity) at UniRef50 scale
  \item Oracle test pattern with statistical utilities (confidence intervals, effect sizes, significance thresholds)
  \item Mutation testing (mutmut) evaluation of the unit test suite's fault-detection capability
  \item Quantitative analysis of the time-vs-coverage trade-off across test layers
\end{itemize}
